# Nasca-RestApi

Микросервис на Flask, предоставляющий REST API для управления списком пользователей.

#Тех Стек
Язык программирования - python 3.12
Framework - Flask
Контейнеризация - Docker, Docker Compose 
## Структура проекта

Nasca-RestApi/
├── app/
│   ├── tests/                # Папка с тестами приложения
│   │   └── test_app.py       # Тесты для проверки эндпоинтов Flask
│   ├── main.py               # Запуск Flask-приложения и эндпоинты
│   ├── requirements.txt      # Зависимости проекта (Flask)
│   ├── Dockerfile            # Инструкция для сборки Docker-образа
│   └── docker-compose.yml    # Конфигурация Docker Compose для запуска
├── scripts/
│   └── server-info.sh        # Скрипт для сбора информации о системе
└── README.md                 # Документация проекта

Все команды по управлению контейнерами выполняются из папки app, так как конфигурационные файлы Docker находятся именно там.
1. Переход в рабочую директорию
cd app

2. Запуск контейнера в фоновом режиме
docker compose up -d

3. Проверка статуса контейнера и здоровья (Health check)
docker compose ps
(В колонке STATUS должно появиться значение Up и healthy).

4. Просмотр логов приложения
docker compose logs -f app

🧪 Запуск тестов
Так как папка tests находится внутри рабочей директории приложения, которая смонтирована в контейнер, запускать тесты можно напрямую внутри изолированного окружения Docker

docker compose exec app python -m unittest tests/test_app.py

📡 Проверка работоспособности (API эндпоинты)
После запуска проект доступен на локальной машине по адресу http://localhost:5000.
Проверка Health Check 
curl http://localhost:5000/health

Основные эндпоинты API

    GET / — Приветственная страница.

    GET /api/users — Получить список всех пользователей.

    GET /api/users/<id> — Получить пользователя по ID.

    POST /api/users — Добавить нового пользователя (передать JSON с полями name и email).

    DELETE /api/users/<id> — Удалить пользователя по ID.

📜 Использование скриптов автоматизации
Перед запуском Bash-скрипта из папки scripts выдайте ему права на выполнение. Из корневой директории проекта (Nasca-RestApi) выполните:
chmod +x scripts/server-info.sh
./scripts/server-info.sh

🛑 Остановка проекта
Чтобы остановить контейнер и очистить ресурсы Docker (сети и контейнеры), выполните:
docker compose down

🛠 Troubleshooting (Решение проблем)
1. Ошибка: Bind for 0.0.0.0:5000 failed: port is already allocated

    Причина: Порт 5000 на вашем компьютере уже занят другим процессом или старым контейнером.

    Решение: Остановите старый контейнер командой docker stop <имя> или полностью очистите окружение этого проекта с помощью:
    Bash

    make compose-down

2. Ошибка при запросе: curl: (7) Failed to connect to localhost port 5000

    Причина: Контейнер упал сразу после старта, либо Flask слушает локальный хост 127.0.0.1 вместо 0.0.0.0 внутри контейнера.

    Решение: Проверьте логи командой make compose-logs. Убедитесь, что в файле app/main.py запуск прописан строго как app.run(host='0.0.0.0', port=5000).

3. Контейнер запускается, но статус в docker compose ps равен unhealthy

    Причина: Docker-контейнер не может достучаться до внутреннего эндпоинта /health во время проверки работоспособности.

    Решение: Проверьте правильность монтирования папок в docker-compose.yml (volumes: - .:/app) и убедитесь, что в контейнере установлена утилита curl (используемый образ python:3.12-alpine поддерживает её выполнение).
